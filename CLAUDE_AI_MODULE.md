# Claude AI — Модуль интеллектуальной поддержки СМИТ

## Обзор

Claude AI — комплексное решение для автоматизации клиентской поддержки интернет-провайдера СМИТ на базе Claude API (Anthropic). Система состоит из двух компонентов:

1. **Модуль FreeScout** — помощник оператора (подсказки, черновики ответов в тикетах)
2. **Django backend** — AI-чат для клиентов (личный кабинет + мобильное приложение)

Оба компонента используют единый API ключ, общую базу знаний и логируют сессии в одну таблицу для централизованного мониторинга.

## Компонент 2: Django Backend (чат для клиентов)

### Назначение
AI-чат в личном кабинете и мобильном приложении. Клиент общается с Claude, который знает его баланс, тариф, адрес и историю.

### Расположение
- **Сервер**: `demo.billing.smit34.ru` (Docker)
- **Путь**: `/opt/carbon-billing/app/`
- **Локально**: `D:\DevTools\Database\2026Carbon\carbon_modern\`

### Файлы

| Файл | Описание |
|------|----------|
| `lk/views/chat.py` | Чат в личном кабинете (POST /lk/chat/send/) |
| `mobile_api/views/chat.py` | Чат в мобильном приложении (POST /mobile-api/v1/chat/message) |
| `billing/services/claude_logger.py` | Логирование сессий в FreeScout DB |
| `lk/templates/lk/chat.html` | Шаблон страницы чата в ЛК |

### Эндпоинты

| Метод | URL | Auth | Описание |
|-------|-----|------|----------|
| GET | `/lk/chat/` | Cookie (ЛК) | Страница чата |
| POST | `/lk/chat/send/` | Cookie (ЛК) | Отправка сообщения |
| POST | `/mobile-api/v1/chat/message` | JWT | Чат из приложения |
| POST | `/mobile-api/v1/chat/escalate` | JWT | Эскалация оператору |
| GET | `/mobile-api/v1/chat/status` | JWT | Статус AI-ассистента |

### Логика обработки сообщения

```
1. Получение сообщения от клиента
2. Проверка быстрых ответов (баланс, тариф) — без API вызова
3. Построение контекста абонента (баланс, тариф, адрес, услуги)
4. Формирование system prompt с правилами
5. Отправка в Claude API с историей диалога (последние 10 сообщений)
6. Фильтрация ответа (убрать "подключить интернет" и т.д.)
7. Проверка на ESCALATE_TO_HUMAN → создание тикета FreeScout
8. Логирование в aida_conversations / aida_messages
9. Возврат ответа клиенту
```

## Логирование сессий

Все AI-сессии (и от операторов, и от клиентов) логируются в единую БД FreeScout.

### Таблицы

**aida_conversations** — сессии:
| Поле | Тип | Описание |
|------|-----|----------|
| session_id | VARCHAR(64) | Уникальный ID сессии |
| started_at | DATETIME | Начало |
| client_name | VARCHAR(255) | ФИО абонента |
| client_phone | VARCHAR(20) | Телефон |
| address | VARCHAR(500) | Адрес |
| tariff | VARCHAR(255) | Тариф |
| utm_source | VARCHAR(100) | `claude_lk` / `claude_mobile` |
| messages_count | INT | Количество сообщений |
| status | ENUM | active / completed / abandoned |
| ticket_created | TINYINT | Была ли эскалация |
| ticket_id | INT | ID тикета FreeScout |

**aida_messages** — сообщения:
| Поле | Тип | Описание |
|------|-----|----------|
| conversation_id | INT | FK на aida_conversations |
| role | ENUM | user / assistant / system |
| content | TEXT | Текст сообщения |
| created_at | DATETIME | Время |

### Просмотр сессий
**URL**: `https://support.smit34.ru/ai-sessions`

Фильтрация по:
- utm_source: `claude_lk`, `claude_mobile`, `aida` (старые)
- Период, статус, наличие эскалации

## System Prompts

### Промпт для клиентов (ЛК / приложение)
Строится динамически в `_build_full_context()`:
- Роль: существующий клиент, УЖЕ подключён
- Данные абонента: баланс, тариф, адрес, номер договора
- Правила: НЕ предлагать подключение, НЕ спрашивать адрес, НЕ выдумывать данные
- При проблемах: диагностика → перезагрузка → тикет
- Для оплаты: направить в раздел «Оплатить»

### Промпт для операторов (FreeScout)
Из файла `Resources/prompts/operator.txt`:
- 21 сценарий S01-S21 с деревьями диагностики
- Проблемные зоны (Червленое, Чапурники, Буревестник)
- Тарифы и цены услуг
- Глоссарий (ONU, LOS, PPPoE, РОБОР)
- Структурированный вывод: Сценарий → Диагноз → Решение → Ответ клиенту → Эскалация

## Устранение неполадок

| Проблема | Решение |
|----------|---------|
| Кнопки AI не появляются в FreeScout | `php artisan freescout:clear-cache --doNotGenerateVars` |
| ЛК чат не отвечает | Проверить `ANTHROPIC_API_KEY` в `.env`, перезапустить web |
| "AI-ассистент отключён" | Установить `CLAUDE_ENABLED_LK=True` в `.env` |
| Сессии не логируются | Проверить `pymysql` установлен, `FREESCOUT_DB_*` настроены |
| Ошибка 401 от Claude | Неверный API ключ — получить новый на console.anthropic.com |
| Медленные ответы | Переключить модель на Haiku 4.5 (самая быстрая) |
