v3.0.0 (build 1404–1444) — июль 2026

Карта абонентов: скорость, права и тёмная тема (build 1741)

  • Карта грузится в 4 раза быстрее: 11,6 → 2,6 с, передача по сети 4585 → 380 КБ. В nginx не был включён gzip для JSON (сжимался только HTML) — теперь сжимаются все JSON-ответы биллинга. Плюс ответ карты больше не экранирует кириллицу: русские буквы занимали по 6 байт вместо двух. Подробнее — Вес карты и скорость.
  • Пустая карта объясняет причину: если у организации нет адресов с точными координатами, вместо молчания выводится счётчик абонентов и ссылка на редактор координат. См. Мультиорганизация.
  • Правка координат дома закрыта правами — раньше сохранить координаты мог любой залогиненный сотрудник. Теперь суперпользователь и группы из ADDRESS_ADMIN_GROUPS.
  • Тёмная тема: вернулся жёлтый индикатор «Приостановлен» в легенде (правило темы гасило любой цвет, начинающийся на «f», — задевало ещё #ffc107 и #fd7e14 по всему проекту), тёмные тайлы получили русские подписи вместо латиницы.
  • Мелочи доступности: подпись маркера для скринридера, кнопка сворачивания легенды выросла с 7×14 до 44×44 px, зум на мобильном — 40×40.

Папки организаций и раскладка абонентов (build 1735–1739)

  • Папку верхнего уровня можно закрепить за организацией — она становится её «домом». Закрепление проходит вглубь: подпапки, абоненты и их финансовые операции получают организацию папки. Иконка красится акцентным цветом компании из брендинга и наследуется вложенными папками. Появилось редактирование папки (имя + организация) — раньше папку можно было только создать или удалить. Подробнее — Папка организации.
  • Раздел «Абоненты» подчиняется выбору организации в шапке: чужие папки скрыты и в дереве, и в содержимом. Прежде папки шли в обход фильтра, поэтому у «СмИТ» висели папки других компаний. Записи без организации видны в любом режиме. Выбор в шапке остаётся фильтром отображения и не влияет на права доступа — см. Мультиорганизация.
  • У тарифа появилась «Директория» — папка, куда попадают его новые абоненты (заняла место неиспользуемой «Валюты»). Список строится от папки организации тарифа с её вложенными. См. Директория тарифа.
  • Раскладка новых абонентов настраивается: по директории тарифа (по умолчанию), только по папке организации или не раскладывать — Настройки → Система → Профиль. Абонент, созданный внутри конкретной папки, остаётся в ней в любом режиме. См. Раскладка новых абонентов.
  • В модалке «Новый абонент» организация выбирается селектом в шапке (в режиме «Все организации»), а списки тарифов и NAS следуют за выбором. Пустой список тарифов объясняется текстом, а NAS показываются все, если своих у компании нет.
  • Папки разовых импортов EBS-импорт <дата> объединены: абоненты перенесены в основную папку, пустые папки удалены. Импорт больше не создаёт папку под каждую дату.

Тарифы по организациям и полноширинный редактор бота (build 1444)

  • В карточке сделки селектор тарифа показывает только тарифы организации сделки: для сделок Робор — тарифы Робор, для СмИТ — тарифы СмИТ. У каждого тарифа теперь обязательно указана организация.
  • Автозаполнение полей лида из расшифровки звонка: тариф (по публичным тарифам с галкой «показывать на сайте»), стоимость подключения, отдельное поле «Настройка», роутер; адрес подключения нормализуется через DaData. Упоминание «робор» в разговоре переводит сделку в организацию Робор со стоимостью подключения 1000 ₽.
  • Канвас редактора Salesbot развёрнут на всю ширину рабочей области; модалка добавления сделки адаптирована для телефонов. Подробнее — Salesbot.

Поддержка: расшифровка и суть звонка (AI)

  • У тикета со звонком — блок «Запись звонка» с плеером и кнопкой «Транскрибировать / Обновить суть»: запись расшифровывается (OpenAI Whisper) и AI выделяет структурную выжимку — суть обращения, категорию, теги, признак «требуется оператор».
  • Тикет автоматически связывается с абонентом по телефону или по номеру договора, названному в разговоре; если у абонента не заполнен телефон — добавляется номер звонящего.
  • Блок доступен и в карточке тикета, и в выезжающей панели (сворачиваемый аккордеон). Тумблер «Авто-транскрибация звонков» в Настройках → Общие запускает разбор автоматически при поступлении звонка. Подробнее — Расшифровка и суть звонка.

Карточка сделки — полноэкранный дровер (build 1441–1442)

  • Сделка открывается одинарным кликом дровером на весь экран: слева «паспорт» с полосой-прогрессом стадий и вкладками Основное / Статистика (все UTM-метки) / Доп. поля / Настроить (+ добавление группы полей), справа — лента с задачами-карточками и композером «Примечание | Задача».
  • Задача требует дату и ответственного; результат добавляет тот, кому она поставлена. Плашка «Возможные дубли» с объединением — прямо в дровере.
  • Доп. телефон контакта: второй номер сам извлекается из комментария заявки; e-mail в шапке, мини-карта адреса, тикеты Поддержки, история стадий, ответы Salesbot и виджеты точки «Карточка сделки» — всё в дровере. Подробнее — Карточка сделки.

Единый поток клиента (build 1439–1440)

  • «Лента клиента» в карточке абонента: обращения, сделки, заметки, задачи и звонки одной хронологией (виджет).
  • Импорт лидов из CSV в канбане — через штатный приём с дедупликацией. Подробнее — Импорт CSV.
  • Наряды монтажника: вид «Наряды» календаря, «Выполнено» двигает сделку в «Подключено» с телефона. Подробнее — Наряды.
  • Дистрибутив: у новых инсталляций с пустой CRM стартовые воронки создаются автоматически (build 1443); «Быстрый старт» — в документации.

Мастер настройки CRM и навигация (build 1436–1438)

  • Мастер настройки (CRM → Настройки → Мастер): 5 шагов — воронки из пресетов, каналы приёма, команда и распределение, AI-бот, виджеты; чеклист «Готовность CRM: N%» из 6 живых проверок.
  • Раздел Настроек «Лицензия и расширения» объединил Лицензию, Модули и Виджеты.
  • Новая сделка создаётся в выбранной воронке (а не по техническому типу); бейдж карточки — имя воронки.
  • Поддержка на виджет-платформе: блоки сайдбара тикета (Биллинг, SLA, Теги, Доп.поля, Учёт времени) включаются из каталога; там же — плитки каналов обращений с тумблером.

NAS Развёртывание (OSS)

Содержание раздела
  • Общая схема работы
  • Под-вкладки
  • Модальное окно вывода команд

Вкладка «Развёртывание» в карточке NAS позволяет управлять конфигурацией MikroTik и другого оборудования. Полностью воспроизводит функционал Carbon Billing 4.

Общая схема работы

1. Инициализация    → rtsh cfg init <scheme>   → создаёт структуру каталогов
2. Заполнение данных → rtsh cfg fill <ip>       → заполняет .ini из биллинга
3. Скачать текущую   → rtsh cfg download        → загружает конфиг с оборудования
4. Генерация новой   → rtsh cfg make            → генерирует .cfg из .ini + .tmplt
5. Загрузка          → rtsh cfg upload          → загружает конфигурацию на NAS
6. Сохранение        → rtsh cfg save            → коммит в git-историю

Под-вкладки

Управление

5 шагов развёртывания + дополнительные команды (status, diff, log, acl upload).

Настройка (INI файлы)

Редактирование .ini файлов конфигурации: main.ini, firewall.ini, netflow.ini, pppoe.ini. Ссылка «Отобразить доступные параметры» показывает все поддерживаемые ключи.

Шаблоны (.tmplt)

Редактирование Jinja2-шаблонов для генерации конфигурации: firewall_address_list.cfg.tmplt, firewall_filter.cfg.tmplt, firewall_nat.cfg.tmplt, netflow.cfg.tmplt, pppoe.cfg.tmplt.

Upload cfg

Файлы сгенерированной конфигурации для загрузки на NAS. Появляются после rtsh cfg make.

Download cfg

Файлы текущей конфигурации, скачанные с оборудования через rtsh cfg download. Только для просмотра.

Модальное окно вывода команд

При нажатии на любую кнопку (rtsh cfg init, fill, make и др.) открывается модальное окно с выводом команды в реальном времени. Кнопки «Скачать вывод» и «Закрыть».
