Поддержка (build 1369–1402)

  • Умный поиск по тикетам: телефон в любом формате, адрес, текст переписки — со сниппетом-подсветкой совпадения в списке.
  • Сайдбар тикета: SLA-таймер «Клиент ждёт ответа» (цвет по порогу), виджет «Прошлые обращения» (5 последних тикетов абонента).
  • Выезжающая панель тикета: мини-Биллинг (аватар/баланс/тариф/статус/соцсети), быстрая смена приоритета, SLA, прошлые обращения.
  • Мобильная свёрнутая sticky-шапка абонента над тредом (аватар/баланс/статус + действия).
  • Вложения: политика типов (multiselect) + лимит объёма, сжатие картинок при загрузке, архивация старых в GCS.
  • Категории обращений пер-ящик, отметка «Перезвонить», блок «тикеты, требующие внимания», кнопка «следующий тикет».
  • Редактирование своего ответа с отметкой о правке; DaData-нормализация адресов; AI авто-триаж старых тикетов.
  • Настройки реорганизованы пер-ящик (вкладки, IMAP+SMTP вместе, Telegram-параметры у ящика), рейтинги, учёт времени.
  • Подробнее — Модуль «Поддержка».

CRM (build 1361–1398)

  • Воронка: компактные колонки, чипы Все/Мои/Наблюдаю/Не назнач., фильтр периода (запоминается), свёрнутые «Подключено»/«Отказ», undo.
  • Дашборд-карта: фильтры/полный экран, тултип с аватаркой и ФИО, цвет маркера по стадии; сайдбар лида для сэйлзов.
  • Рекламные кампании: UTM-линк-билдер + QR, атрибуция сделок, дашборд CAC.
  • Подробнее — CRM — продажи и подключения.

Внутренний чат (build 1403)

  • Toast-уведомления о новых личных/групповых сообщениях: матовое стекло, аватар, имя, время, «Ответить»; мобильная адаптация.
  • Отчёты по расписанию в чат: параметризованные SQL-отчёты работают — даты авто-заполняются периодом по частоте.
  • Подробнее — Внутренний чат.

Абоненты и данные (build 1398–1403)

  • Аватарки и соцсети (VK/TG/MAX) в профиле, drawer-сайдбаре, тикетах, карте CRM; импорт из FreeScout с fuzzy-матчингом ФИО (транслитерация, Ё→Е).
  • Нормализация телефонов к E.164 (fix_phone_numbers): 268 номеров, каждый — с записью в аудит. +77XXXXXXXXX — валидный формат, не задвоение.
  • СОРМ: серия/номер паспорта принимают буквы (иностранные граждане). Смена адреса абонента больше не откатывается.

Интерфейс и инфраструктура

  • Переключатель темы Система/Светлая/Тёмная в меню профиля; dirty-save кнопка и i-иконки-popover во всех настройках; 6 стилей вкладок; единые блоки-карточки.
  • FreeRADIUS: фикс BlastRADIUS (Message-Authenticator в Access-Accept — новые MikroTik пускают абонентов), авто-перезагрузка при добавлении NAS, часовой logrotate.
  • GCS: managed-режим через шлюз сервера лицензий (квоты, постоянные публичные URL), приоритет собственных ключей.

v2.1.0 (build 996–997) — 6 июня 2026

Инструменты самодиагностики связи с HelpDesk в настройках интеграции и восстановление вкладки «Маркетинг» в карточке абонента.

Диагностика HelpDesk в настройках (build 996)

  • «Проверить БД» с подсказкой по доступу — при ошибке подключения к базе HelpDesk (доступ закрыт для IP сервера биллинга) инструмент показывает понятное объяснение и готовую команду для администратора с уже подставленным исходящим IP сервера. Это частая поломка после переезда сервера биллинга на новый адрес.
  • «Проверка поиска клиента» — новый блок на вкладке Настройки → Интеграции → FreeScout: по номеру договора показывает, найден ли абонент, каким способом сопоставился клиент HelpDesk (договор / email / телефон / имя), его ID и число тикетов. Позволяет диагностировать «тикеты не отображаются» без обращения к администратору сервера. Подробнее — HelpDesk → Диагностика связи.
  • Приоритет поиска клиента уточнён — реальный email теперь предпочтительнее телефона (телефон часто общий у нескольких абонентов), а среди совпадений по телефону выбирается карточка с настоящим email, а не служебная заглушка. Устраняет показ «чужой» пустой карточки без тикетов.

Вкладка «Маркетинг» (build 997)

  • Восстановлено открытие вкладки «Маркетинг» в карточке абонента и её адрес приведён к виду /admin/Abonents/<id>/marketing/ (раньше вела на адрес другой вкладки). Вкладка получила постоянный идентификатор вместо «плавающего», из-за которого периодически переставала открываться.

v2.1.0 (build 864–885) — 18 мая 2026

Доработки логики разблокировок (фин-блок, b_sys, b_own), 3 read-only audit-команды, backup-модуль self-contained, синхронизация с EBS suspendedperiod, UI настроек профиля абонента (новая вкладка с 33 настройками в 2-колоночном layout).

Логика разблокировки и фин-блоки

  • auto-unblock b_negbal в _credit_abonent (build 868) — после webhook-оплаты (YooKassa/W1/Generic) фин-блокировка снимается автоматически. Заменяет ручную проводку оператора.
  • signals.sync_tarif_on_create для NULL→X (build 877, 1A) — через sentinel-маркер триггерит копирование шаблона UsersUsluga при первом назначении тарифа. Раньше шаблон не подтягивался — оператор менял тариф на другой и обратно.
  • auto-unblock b_sys «не назначен тариф» (build 877, 1B) — при появлении tarif_id блок снимается сразу, без отдельного нажатия «снять блок».
  • auto-unblock b_negbal в promise_sync + EBS reconcile (build 877, 2A) — симметрично 868 для всех путей зачисления баланса (webhook + EBS-pull).
  • Celery beat safety net (build 877, 2C) — auto-unblock-bnegbal-with-positive каждые 5 мин ловит любые остаточные случаи + Telegram-алёрт если >5 за тик.
  • Кнопка 💰 «Снять фин-блок» (build 877, 2E) — в шапке карточки абонента при активном b_negbal. Url /admin/abonents/force_unblock_negbal/<id>/. Баланс не меняется.
  • sync_ebs_suspendedperiod + beat + кнопка 🏖 (build 884) — синхронизация добровольных блокировок с EBS. Снимает зомби-блоки из миграционного импорта, когда отпуск в EBS закрыт. Кнопка в шапке для ручного снятия.

Аудит и safety net

  • 3 read-only audit-команды (build 870) — audit_block_consistency (4 проверки блокировок+тарифов), audit_duplicates (FinOps/MsgStack/UsersUsluga), audit_billing_consistency (сверка AdminAccounts vs SUM(FinOps)).
  • Фикс дублей FinOps в promise_sync (build 872) — broad-dedup в окне ±60s по abonent+op_type+summa.
  • Ежедневный audit_block_consistency_daily (build 873/875) — Celery beat в 06:00 UTC, Telegram-алёрт при расхождениях.
  • disable_abonents_without_tariff (build 871) — массовая блокировка абонентов без тарифа.

Backup-модуль (build 876)

  • _make_docker_backup теперь включает Dockerfile, docker-compose.prod.yml, .env, requirements.txt, manage.py, gunicorn.conf.py и папку docker/ — полный self-contained пакет для разворачивания на пустом VPS.
  • Bind-mount /opt/smit-billing/app:/host_app:ro в web и celery контейнерах.
  • restore.sh переписан: принимает TARGET_DIR, sanity-check на database.dump+manage.py, подтверждение перед перезаписом, корректно поднимает БД, восстанавливает дамп, билдит и запускает контейнеры.

UI — настройки и виджеты

  • Новая вкладка /admin/settings/system/?tab=abonent_profile (build 879–883) — 33 настройки в 9 секциях: Quick Cards, видимость секций карточки, колонки списка, AJAX/refresh, дефолты карточки, drag&drop вкладок + 4 стиля, баннеры уведомлений, прочее.
  • 2-колоночный layout для вкладки «Профиль» (build 885) — CSS column-count:2 на ≥1280px, секции укладываются masonry-style плотно. На узких — 1 колонка автоматически.
  • Переименование вкладок (build 885): «Производительность» → «Скорость», «Сессии и RADIUS» → «RADIUS».
  • Виджет «Абоненты NAS» — общий онлайн (build 882) — на дашборде в шапке виджета теперь сумма активных RADIUS-сессий по всем NAS.

Соблюдённые правила

  • EBS — источник правды по балансам. Поле ostatok не правится никакими скриптами или командами.
  • Все блокировки/разблокировки — через AbonentsBlock (наша таблица, EBS её не трогает).
  • Аудит на каждое действие через billing.services.audit_log.write_audit.
  • Toggle через VpnConst для отключения автоматики при инцидентах: EBS_PROMISE_SYNC_ENABLED, AUTO_UNBLOCK_BNEGBAL_ENABLED, EBS_SUSPEND_SYNC_ENABLED.

v2.0.0 (build 838–861) — 15 мая 2026

Крупный рестайлинг карточки абонента и единая дизайн-система компонентов. Версия повышена до 2.0 — пользовательский интерфейс существенно перестроен.

Карточка абонента — рестайлинг (build 838–860)

  • Hero-блок тарифа (build 838) — крупный блок с иконкой-плиткой, названием тарифа, ценой/мес и кнопкой «Изменить тариф». Заменил скучную строку поля.
  • Объединение «Договор и контакты» (build 843) — subgroup 26 «Договор» + 61 «Контактная информация» в единый блок. Юр-поля (Отправлять счёт / Формировать акт / Дата выставления акта) показываются динамически по чекбоксу «Юр. лицо».
  • Скрыто: ФИО-заголовок в форме (дублировал шапку), поле «Точка подключения» (есть отдельная вкладка), «Группа» (parent), «Скидка %» (рекомендуется лояльность), весь блок «Особые настройки» (17 технических полей).
  • Подзаголовки в «Смена тарифа / Блокировки» (build 846) — три смысловые группы: Запланированная смена / Добровольная блокировка / Административная блокировка.

NAS Развёртывание (OSS)

Содержание раздела
  • Общая схема работы
  • Под-вкладки
  • Модальное окно вывода команд

Вкладка «Развёртывание» в карточке NAS позволяет управлять конфигурацией MikroTik и другого оборудования. Полностью воспроизводит функционал Carbon Billing 4.

Общая схема работы

1. Инициализация    → rtsh cfg init <scheme>   → создаёт структуру каталогов
2. Заполнение данных → rtsh cfg fill <ip>       → заполняет .ini из биллинга
3. Скачать текущую   → rtsh cfg download        → загружает конфиг с оборудования
4. Генерация новой   → rtsh cfg make            → генерирует .cfg из .ini + .tmplt
5. Загрузка          → rtsh cfg upload          → загружает конфигурацию на NAS
6. Сохранение        → rtsh cfg save            → коммит в git-историю

Под-вкладки

Управление

5 шагов развёртывания + дополнительные команды (status, diff, log, acl upload).

Настройка (INI файлы)

Редактирование .ini файлов конфигурации: main.ini, firewall.ini, netflow.ini, pppoe.ini. Ссылка «Отобразить доступные параметры» показывает все поддерживаемые ключи.

Шаблоны (.tmplt)

Редактирование Jinja2-шаблонов для генерации конфигурации: firewall_address_list.cfg.tmplt, firewall_filter.cfg.tmplt, firewall_nat.cfg.tmplt, netflow.cfg.tmplt, pppoe.cfg.tmplt.

Upload cfg

Файлы сгенерированной конфигурации для загрузки на NAS. Появляются после rtsh cfg make.

Download cfg

Файлы текущей конфигурации, скачанные с оборудования через rtsh cfg download. Только для просмотра.

Модальное окно вывода команд

При нажатии на любую кнопку (rtsh cfg init, fill, make и др.) открывается модальное окно с выводом команды в реальном времени. Кнопки «Скачать вывод» и «Закрыть».
